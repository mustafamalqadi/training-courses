ما هِيَ السَّحابة الخاصّة الحَديثة؟ — What is Modern Private Cloud?

Private Cloud is NOT a Location — It's an Operating Model.

السَّحابة الخاصّة لَيسَت مَكاناً — إِنَّها نَموذَج تَشغيلي. طَريقة عَمَل تُقَدِّم مُرونة وَأَتمَتة وَقابِليّة تَوَسُّع السَّحابة مَع الحِفاظ على التَّحَكُّم في البَيانات والأَمان والمُرونة والامتِثال.

🧠 Five Key Attributes — خَمس سِمات أَساسيّة لِلسَّحابة الخاصّة

  • Software-Defined Infrastructure — بِنية تَحتيّة مُعَرَّفة بِالبَرمَجيّات (Compute, Storage, Network, Security).
  • Self-Service & Automation — خِدمة ذاتيّة مَع أَتمَتة وَتَحَكُّم لِلمُستَأجِرين.
  • Workload Agnostic — دَعم أَحمال العَمَل الحَديثة والتَّقليديّة والذَّكاء الاصطِناعي (VMs + Containers/K8s).
  • Location Agnostic — مَحَلّي، سَحابي، حَوافّ، سِيادي — حَصري لِمُؤسَّسة واحِدة.
  • Security & Governance — مُرونة، تَحَكُّم في التَّكلِفة، حَوكَمة آليّة.

  • تَرخيص ثابِت — بَدَلاً مِن نَموذَج الدَّفع حَسَب الاستِخدام.

ما هُوَ VMware Cloud Foundation؟

VMware Cloud Foundation (VCF) — مَنصّة سَحابة خاصّة (Private Cloud Platform) تُوَفِّر مَنصّة سَحابيّة مُتَكامِلة بِالكامِل، مُؤَتمَتة، قابِلة لِلتَّوسيع، وَمَبنيّة على خَدَمات مُعَرَّفة بِالبَرمَجيّات لِلـ Compute وَStorage وَNetworking وَSecurity وَCloud Management.

VCF VMware Cloud Foundation: Key Use Cases

VCF يَدعَم حالات استِخدام مُتَنَوِّعة تَشمَل:

  • Private Cloud Platform — مَنصّة سَحابة خاصّة مُتَكامِلة.
  • Modern Application Platform — دَعم Kubernetes وَالتَّطبيقات الحَديثة.
  • AI/ML Infrastructure — بِنية تَحتيّة لِلذَّكاء الاصطِناعي.
  • Hybrid Cloud — تَكامُل مَع السُّحُب العامّة.
  • Edge Computing — حَوسَبة الحَوافّ في المَواقِع المُوَزَّعة.

مُكَوِّنات VCF — Core Components

C Compute — الحَوسَبة

vSphere يُوَفِّر مَنصّة الحَوسَبة الافتِراضيّة الأَساسيّة لِـ VMware Cloud Foundation:

  • ESX — مَنصّة الحَوسَبة الَّتي تُنشِئ وَتُشَغِّل الـ VMs وَأَحمال العَمَل الأُخرى.
  • vCenter — خِدمة إِدارة مُضيفات ESX المُتَعَدِّدة مَع إِمكانيّات vMotion وَHigh Availability.
S Storage — التَّخزين

VMware Cloud Foundation يُوَفِّر خَيارات تَخزين مَرِنة لِنَشر مَثيل VCF جَديد. التَّخزين الأَساسي يَعتَمِد على vSAN كَحَل HCI (Hyper-Converged Infrastructure) مُدمَج.

N Networking — الشَّبَكات

NSX يُوَفِّر الشَّبَكات المُعَرَّفة بِالبَرمَجيّات (Software-Defined Networking) بِما في ذَلِك:

  • Overlay Networks — شَبَكات افتِراضيّة مُنفَصِلة عَن الفيزيائيّة.
  • Micro-Segmentation — تَجزِئة دَقيقة لِلأَمان.
  • Distributed Firewall — جِدار ناري مُوَزَّع.
  • Load Balancing — مُوازَنة الأَحمال.
O Operations — العَمَلِيّات

VCF Operations يُوَفِّر لِلمُسؤولين تَجرُبة مُوَحَّدة وَغَنيّة بِالمَيزات لإِدارة الأُسطول (Fleet Management):

  • إِدارة الهُوِيّة وَالوُصول (Identity & Access).
  • إِدارة الشَّهادات وَكَلِمات المُرور.
  • إِدارة دَورة الحَياة (Lifecycle Management).
  • إِدارة التَّكوين وَكَشف الانحِرافات (Configuration Drift).
A Automation — الأَتمَتة

VCF Automation يُقَدِّم سَحابة خاصّة بِخِدمة ذاتيّة لِتَطبيقات AI وَK8s وَVM:

  • Self-Service Catalog — كَتالوج خِدمة ذاتيّة لِلنَّشر.
  • IaaS — بِنية تَحتيّة كَخِدمة.
  • Template Designer — تَصميم قَوالِب النَّشر.
  • Workflow Orchestration — تَنسيق سَير العَمَل.

vSphere Supervisor — خَدَمات أَساسيّة

vSphere Supervisor يُقَدِّم خَدَمات IaaS لِلسَّحابة الخاصّة مُباشَرَةً وَيَتَوَسَّع بِخَدَمات قابِلة لِلتَّمديد لِزيادة رِضا المُطَوِّرين. يَشمَل ذَلِك إِدارة VMs وَContainers وَKubernetes Clusters مِن مَنصّة واحِدة.

الأَمان في كُلّ طَبَقة — Security at Every Layer

🔒 Security at Every Layer of Cloud Foundation

VCF يُوَفِّر أَماناً مُتَكامِلاً عَبر جَميع الطَّبَقات:

  • Compute Layer — أَمان على مُستَوى الـ Hypervisor وَالـ VMs.
  • Network Layer — Micro-Segmentation وَDistributed Firewall مَع NSX.
  • Storage Layer — تَشفير البَيانات في الراحة وَأَثناء النَّقل مَع vSAN.
  • Management Layer — إِدارة مَركَزِيّة لِلشَّهادات وَكَلِمات المُرور.

  • نَشر vCenter Appliance على أَوَّل مُضيف ESX في نِطاق الإِدارة.

  • إِضافة جَميع مُضيفات ESX في نِطاق الإِدارة.
  • إِنشاء vSphere Distributed Switch لِجَميع المُضيفات.
  • تَرحيل VMs إِلى VM Management distributed port group.
  • تَرحيل vmknics الإِداريّة إِلى vDS.
  • تَوسيع vSAN Datastore لِيَشمَل جَميع المُضيفات.
  • تَكوين HA وَDRS في الكَتَلَسْتِر.

  • إِضافة vCenter كَـ Compute Manager.
  • إِنشاء Overlay Transport Zone.
  • إِنشاء TEP IP Pool.
  • إِنشاء NSX Uplink Profile لِلمُضيفات.
  • إِنشاء Transport Node Profile وَتَطبيقه على جَميع المُضيفات.

  2. VCF Automation.

عَرض أَبسَط يَتَضَمَّن:

❌ Before VCF 9.0
  • مِفتاح لِكُلّ مُكَوِّن — عَدَد كَبير مِن مَفاتيح التَّرخيص.
  • مَفاتيح خاصّة بِالإِصدار — مَثَل vSphere 7 مُقابِل vSphere 8.
  • رُؤية مَحدودة — vCenter واحِد في كُلّ مَرّة.
  • تَتَبُّع يَدَوي — جَداوِل بَيانات قَد تُؤَدّي لِتَراخيص غَير مُستَخدَمة.
  • تَعقيد التَّقسيم وَالدَّمج — عِند تَغيير البِنية التَّحتيّة.
✅ VCF 9.0 Solves
  • مِلَفّ تَرخيص واحِد — (Core, TiB).
  • لا مَفاتيح خاصّة بِالإِصدار — لا حاجَة لِمِفتاح جَديد مِن v8 → v9.
  • رُؤية مُوَحَّدة — عَبر VCF Operations.
  • عَرض مُجَمَّع لِلاستِهلاك — بَدَل الجَداوِل.
  • تَعديل ديناميكي لِلكَمِّيّات — مُرونة عِند التَّوَسُّع.

إِدارة الأُسطول — Fleet Management Functions

💡 سِت وَظائف رَئيسيّة لإِدارة الأُسطول

VCF Operations Fleet Management يُقَدِّم تَجرُبة عَمَلِيّات مُوَحَّدة وَغَنيّة تَمتَد عَبر الوَظائف التّالية:

🔑 1. Identity & Access — SSO

VCF Single Sign-On يُوَفِّر مُصادَقة مُوَحَّدة لِلوُصول إِلى مُكَوِّنات VCF المُتَعَدِّدة بِمَجموعة بَيانات اعتِماد واحِدة:

  • تَسجيل دُخول سَلِس عَبر جَميع مُكَوِّنات VCF.
  • إِدارة هُوِيّة على مُستَوى الأُسطول مِن VCF Operations.
  • دَعم IDPs خارِجيّة: Okta, Microsoft EntraID, ADFS, Ping, Generic SAML 2.0.
  • مَصادِر هُوِيّة: Active Directory / Open LDAP عَبر OIDC, SAML, LDAP.

يَتِم نَشر VCF Identity Broker Appliance عَبر Fleet Management.

📜 2. Certificate Management — الشَّهادات

إِدارة مَركَزِيّة لِلشَّهادات عَبر مُكَوِّنات البِنية التَّحتيّة وَالإِدارة في واجِهة واحِدة:

  • Central Management — إِدارة مَركَزِيّة لِجَميع الشَّهادات.
  • Nondisruptive Replacement — استِبدال بِدون انقِطاع.
  • Multiple CAs — دَعم سُلُطات شَهادات مُتَعَدِّدة.
  • Expiry Notifications — تَنبيهات انتِهاء الصَّلاحيّة.
  • Automatic Renewal — تَجديد آلي مَع VMware CA وَMicrosoft CA.
🔐 3. Password Management — كَلِمات المُرور

إِدارة مَركَزِيّة لِكَلِمات مُرور root وَlocal admin لِكُلّ مُكَوِّن VCF:

  • تَوحيد إِدارة كَلِمات المُرور.
  • لَوحة تَحَكُّم مُوَحَّدة لِلإِدارة وَالإِشعارات.
  • مَعايير مُوَحَّدة: التَّعقيد، تَكرار التَّدوير، فَترة الإِشعار.
🏷️ 4. Tag Management — العَلامات

إِدارة مَركَزِيّة لِلعَلامات عَبر مُكَوِّنات VCF المُتَعَدِّدة:

  • تَطبيق العَلامات على عِدّة مُكَوِّنات VCF في آنٍ واحِد.
  • تَناسُق عَبر NSX, vCenter, VCF Automation.
  • تَعيين العَلامات على نِطاق واسِع.
  • استِيراد العَلامات مِن مَثيلات vCenter الحاليّة مَع حَلّ التَّعارُضات.

VPC Architecture — هَندَسة الشَّبَكة الخاصّة الافتِراضيّة

🧠 مُكَوِّنات VPC

  • VPC Gateway (VPC GW) — بَوّابة يَتَّصِل بِها كُلّ حِمل عَمَل لِلاتِّصال الخارِجي.
  • Transit Gateway (TGW) — تَوجيه مُوَزَّع بَين VPCs المُتَّصِلة داخِل مَشروع.
  • External Connectivity — اتِّصال خارِجي عَبر VLAN أَو Provider Gateway (Tier-0).
  • Subnets — شَبَكات مَنطِقيّة داخِل VPC يَتَّصِل بِها أَحمال العَمَل.

Correct: C — سِياسات الحَوكَمة تَشمَل Quotas, Leases, Approvals, وَIaaS Policies. تَحديث Firmware لَيسَ مِن سِياسات الحَوكَمة.
Q27 VMware Live Recovery provides what percentage faster downtime resolution?
Correct: C — Live Recovery يُوَفِّر 75% أَسرَع في حَلّ فَترات التَّوَقُّف (IDC مارس 2024).
Q28 vDefend Firewall with ATP adds which capability over basic vDefend?
Correct: B — ATP يُضيف: IDS/IPS, Malware Prevention, NTA, وَNDR. الباقي مَوجود في النُّسخة الأَساسيّة.
Q29 VMware Data Services Manager supports which database operations?
Correct: B — DSM يَدعَم: Provision, Backup, Clone, Patch, Scale, Cluster.
Q30 Avi Load Balancer reduces devices to manage by:
Correct: C — Avi Load Balancer يُقَلِّل الأَجهِزة لِلإِدارة بِمِقدار 10x (عَشر مَرّات أَقَل).
Q31 VCF security is implemented at which layers?
Correct: B — أَمان VCF مُتَكامِل عَبر: Compute Layer, Network Layer, Storage Layer, وَManagement Layer.
Q32 The "Organization Administrator" role in VCF is responsible for:
Correct: B — Organization Administrator يُدير مَوارِد المُؤسَّسة وَالمَشاريع وَالسِّياسات.
Q33 How does VCF 9.0 handle license keys when upgrading from v8 to v9?
Correct: B — في VCF 9.0، لا حاجَة لِمِفتاح تَرخيص جَديد عِند التَّرقية مِن v8 إِلى v9 — التَّراخيص لَيسَت خاصّة بِالإِصدار.
Q34 VCF Operations password management standardizes which aspects?
Correct: B — إِدارة كَلِمات المُرور تُوَحِّد: التَّعقيد (Complexity), تَكرار التَّدوير (Rotation Frequency), وَفَترة الإِشعار (Notification Period).
Q35 During vSphere Cluster configuration, the VCF Installer creates:
Correct: B — أَثناء تَكوين vSphere Cluster: إِنشاء vDS، تَوسيع vSAN، تَكوين HA وَDRS.
Q36 The three strategic pillars for private cloud use cases are:
Correct: B — الرَّكائز الثَّلاث: تَحديث البِنية التَّحتيّة، تَحديث التَّطبيقات، وَتَحديث الأَمان.
Q37 The Developer Center page in VCF Operations provides:
Correct: B — Developer Center يُوَفِّر APIs, CLI libraries, وَأَداة Management Pack Builder (بِدون كود) لِبِناء حُزَم إِدارة مُخَصَّصة.
Q38 VMware Private AI Foundation requires purchasing which additional component directly from NVIDIA?
Correct: B — تَرخيص NVIDIA AI Enterprise يُشتَرى مُنفَصِلاً وَمُباشَرَةً مِن NVIDIA.
Q39 In the VCF 5.2 → 9.0 upgrade Phase 1, which components are upgraded first?
Correct: B — في Phase 1 مِن تَرقية VCF 5.2 → 9.0: تَرقية مُكَوِّنات Aria إِلى VCF Fleet 9.0 Management Component.
Q40 The license warning in VCF 9.0 begins at ___ days before expiration.
Correct: A — مُدّة التَّجديد 180 يَوم. التَّحذير يَبدَأ عِند 150 يَوم (أَي مُتَبَقّي 30 يَوم لِلانتِهاء). بَعد الانتِهاء: 90 يَوم لِلامتِثال.
