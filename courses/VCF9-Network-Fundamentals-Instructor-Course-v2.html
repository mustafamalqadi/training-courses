ЩӮЩҺШЁЩ„ ШЈЩҺЩҶ ШӘЩҺШәЩҲШө ЩҒЩҠ NSX ЩҲЩҺOverlay ЩҲЩҺVPCШҢ ШӘЩҺШӯШӘШ§Ш¬ ШЈЩҺЩҶ ШӘЩҺЩҒЩҮЩҺЩ… Щ„ЩҗЩ…Ш§Ш°Ш§ ЩҲЩҸШ¬ЩҗШҜЩҺШӘ Ш§Щ„ШҙЩҺЩ‘ШЁЩҺЩғШ§ШӘ Ш§Щ„Щ…ЩҸШ№ЩҺШұЩҺЩ‘ЩҒШ© ШЁЩҗШ§Щ„ШЁЩҺШұЩ…ЩҺШ¬ЩҠЩ‘Ш§ШӘ. ЩҮЩҺШ°Ш§ Ш§Щ„ЩӮЩҗШіЩ… ЩҠЩҸШӨЩҺШіЩҗЩ‘Ші Ш§Щ„ЩҒЩҺЩҮЩ… Ш§Щ„Щ…ЩҗШ№Щ…Ш§ШұЩҠ Ш§Щ„ЩҺЩ‘Ш°ЩҠ ЩҠЩҺЩҒШөЩҗЩ„ Ш§Щ„Щ…ЩҸЩҮЩҺЩҶШҜЩҗШі Ш§Щ„Щ…ЩҸШӯШӘЩҺШұЩҗЩҒ Ш№ЩҺЩҶ Щ…ЩҸШ¬ЩҺШұЩҺЩ‘ШҜ Щ…ЩҸЩҶЩҺЩҒЩҗЩ‘Ш° Ш§Щ„ШЈЩҺЩҲШ§Щ…ЩҗШұ.

0.5.6 Knowledge Check

Q1. What is the primary purpose of SDN (Software-Defined Networking)?

Correct: B

SDN moves network functions (switching, routing, firewalling) to software while still relying on physical infrastructure as the underlay.

Q2. Which encapsulation protocol does NSX 9.0 use for overlay traffic?

Correct: C

NSX uses Geneve (Generic Network Virtualization Encapsulation) on UDP port 6081.

Q3. What does the term "Transport Node" refer to in NSX?

Correct: B

Transport Nodes are ESX Hosts or NSX Edge Nodes that have been prepared for NSX overlay networking.

ЩҒЩҠ ШЁЩҠШҰШ§ШӘ Ш§Щ„ШҘЩҗЩҶШӘШ§Ш¬ШҢ ЩҠЩҸЩҲШөЩү ШЁЩҗЩҶЩҺШҙШұ 3-node cluster. Щ„ЩҺЩғЩҗЩҶ VCF 9.0 ЩҠЩҺШҜШ№ЩҺЩ… ЩҶЩҺШҙШұ NSX Manager ЩҲШ§ШӯЩҗШҜ Щ„ЩҗШЁЩҠШҰШ§ШӘ Ш§Щ„Ш§ЩҗШ®ШӘЩҗШЁШ§Шұ ШЈЩҺЩҲ Ш№ЩҗЩҶШҜЩҺЩ…Ш§ Щ„Ш§ ЩҠЩҸШіШӘЩҺШ®ШҜЩҺЩ… NSX.

1.3 Virtual IP Address вҖ” Ш№ЩҸЩҶЩҲШ§ЩҶ IP Ш§Щ„Ш§ЩҗЩҒШӘЩҗШұШ§Ш¶ЩҠ

Ш№ЩҗЩҶШҜЩҺЩ…Ш§ ЩҠЩҸЩҶШҙЩҺШұ NSX Management Cluster ЩҒЩҠ ЩҲЩҺШ¶Ш№ Ш§Щ„ШӘЩҺЩ‘ЩҲШ§ЩҒЩҸШұ Ш§Щ„Ш№Ш§Щ„ЩҠ:

рҹ§  VIP вҖ” ЩғЩҺЩҠЩҒ ЩҠЩҺШ№Щ…ЩҺЩ„

  • ЩғЩҸЩ„Щ‘ Manager Nodes Ш№ЩҺЩ„Щү ЩҶЩҺЩҒШі VLAN/Subnet.
  • ЩҠЩҸЩҶШӘЩҺШ®ЩҺШЁ node ЩҲШ§ШӯЩҗШҜ ЩғЩҺЩҖ Leader.
  • Virtual IP ЩҠЩҺШӘЩҺЩ‘ШөЩҗЩ„ ШЁЩҗШ§Щ„ЩҖ Leader.
  • Щ„Ш§ ЩҠЩҲШ¬ЩҺШҜ load balancing Ш№ЩҺШЁШұ Ш§Щ„ЩҖ Managers ШЁЩҗШ§ШіШӘЩҗШ®ШҜШ§Щ… VIP.
  • ШҘЩҗШ°Ш§ ЩҒЩҺШҙЩҺЩ„ Ш§Щ„ЩҖ Leader вҶ’ Ш§ЩҗЩҶШӘЩҗШ®Ш§ШЁ Ш¬ЩҺШҜЩҠШҜ вҶ’ GARP вҶ’ VIP ЩҠЩҺЩҶШӘЩҺЩӮЩҗЩ„ Щ„ЩҗЩ„ЩҖ Leader Ш§Щ„Ш¬ЩҺШҜЩҠШҜ.

рҹ’Ў ЩҶЩҺШөЩҠШӯШ© Ш№ЩҺЩ…ЩҺЩ„ЩҠЩ‘Ш©

ШҘЩҗШ°Ш§ ЩҒЩҺЩӮЩҺШҜШӘ Ш§Щ„Ш§ЩҗШӘЩ‘ШөШ§Щ„ ШЁЩҗШ§Щ„ЩҖ VIPШҢ Ш¬ЩҺШұЩҗЩ‘ШЁ Ш§Щ„Ш§ЩҗШӘЩ‘ШөШ§Щ„ Щ…ЩҸШЁШ§ШҙЩҺШұШ© ШЁЩҗШЈЩҺШӯЩҺШҜ Ш§Щ„ЩҖ Manager nodes. ШҘЩҗШ°Ш§ ЩҶЩҺШ¬ЩҺШӯ вҶ’ Ш§Щ„Щ…ЩҸШҙЩғЩҗЩ„Ш© ЩҒЩҠ VIP election. ШҘЩҗШ°Ш§ ЩҒЩҺШҙЩҺЩ„ вҶ’ Ш§Щ„Щ…ЩҸШҙЩғЩҗЩ„Ш© ЩҒЩҠ Ш§Щ„ЩҖ Manager ЩҶЩҺЩҒШіЩҮ ШЈЩҺЩҲ Ш§Щ„ШҙЩҺЩ‘ШЁЩҺЩғШ©.

рҹ§  CCP vs LCP

  • CCP (Central Control Plane): Ш¬ЩҸШІШЎ Щ…ЩҗЩҶ NSX Manager. ЩҠЩҺШӯШіЩҗШЁ Runtime State. ЩҠЩҸЩҲЩҺШІЩҗЩ‘Ш№ Ш§Щ„Щ…ЩҺШ№Щ„ЩҲЩ…Ш§ШӘ Ш№ЩҺШЁШұ LCP.
  • LCP (Local Control Plane): Щ…ЩҺЩҲШ¬ЩҲШҜ Ш№ЩҺЩ„Щү ЩғЩҸЩ„Щ‘ Transport Node (Host ШЈЩҺЩҲ Edge). ЩҠЩҸШұШ§ЩӮЩҗШЁ ШӯШ§Щ„Ш© Ш§Щ„ШұЩҺЩ‘ЩҲШ§ШЁЩҗШ· Ш§Щ„Щ…ЩҺШӯЩҺЩ„Щ‘ЩҠЩ‘Ш©. ЩҠЩҺШҜЩҒЩҺШ№ Ш§Щ„ШӘЩҺЩ‘ЩҮЩҠШҰШ§ШӘ Щ„ЩҗЩ…ЩҸШӯЩҺШұЩҗЩ‘ЩғШ§ШӘ Ш§Щ„ШӘЩҺЩ‘ЩҲШ¬ЩҠЩҮ.

  4. LCP: Ш№ЩҺЩ„Щү ЩғЩҸЩ„Щ‘ Transport NodeШҢ ЩҠЩҸШ·ЩҺШЁЩҗЩ‘ЩӮ Ш§Щ„ШӘЩҺЩ‘ЩҮЩҠШҰШ© Ш№ЩҺЩ„Щү VDS/N-VDS.

6.4 Micro-segmentation вҖ” Ш§Щ„ШӘЩҺЩ‘Ш¬ШІЩҗШҰШ© Ш§Щ„ШҜЩҺЩ‘ЩӮЩҠЩӮШ©

Micro-segmentation = ШӘЩҺШ·ШЁЩҠЩӮ firewall rules Ш№ЩҺЩ„Щү Щ…ЩҸШіШӘЩҺЩҲЩү Ш§Щ„ЩҖ VM Ш§Щ„ЩҒЩҺШұШҜЩҠ вҖ” ШЁЩҺШҜЩҺЩ„ subnet ШЈЩҺЩҲ VLAN. ЩғЩҸЩ„Щ‘ VM Щ„ЩҺЩҮШ§ policy Ш®Ш§ШөЩ‘Ш© ШЁЩҗЩҶШ§ШЎЩӢ Ш№ЩҺЩ„Щү Ш§Щ„ЩҖ roleШҢ Ш§Щ„ШӘЩҺЩ‘Ш·ШЁЩҠЩӮШҢ Ш§Щ„ШЁЩҠШҰШ©ШҢ ШЈЩҺЩҲ ШЈЩҺЩҠЩ‘ metadata.

  1. ШӘЩҺШӯШҜЩҠШҜ Ш§Щ„ЩҖ Groups: ШөЩҺЩҶЩҗЩ‘ЩҒ VMs ШӯЩҺШіЩҺШЁ: NameШҢ TagШҢ OSШҢ SegmentШҢ ШЈЩҺЩҲ Membership criteria.
  2. ШӘЩҺШ№ШұЩҠЩҒ Ш§Щ„ЩҖ Policies: ШЈЩҺЩҶШҙЩҗШҰ DFW Section Щ„ЩҗЩғЩҸЩ„Щ‘ ШӘЩҺШ·ШЁЩҠЩӮ ШЈЩҺЩҲ ШЁЩҠШҰШ©.
  3. ЩғЩҗШӘШ§ШЁШ© Ш§Щ„ЩҖ Rules: Source Group вҶ’ Destination Group вҶ’ Service (Port/Protocol) вҶ’ Action (Allow/Drop/Reject).
  4. Default Rule: Ш§ЩҗШ¬Ш№ЩҺЩ„ Ш§Щ„ЩҖ default rule = Drop вҖ” ЩҒЩҺЩӮЩҺШ· Ш§Щ„ЩҖ traffic Ш§Щ„Щ…ЩҸШөЩҺШұЩҺЩ‘Шӯ ШЁЩҗЩҮ ЩҠЩҺЩ…ЩҸШұЩ‘ (Zero-Trust).
  5. Ш§Щ„Щ…ЩҸШұШ§ЩӮЩҺШЁШ©: Ш§ЩҗШіШӘЩҺШ®ШҜЩҗЩ… Flow Monitoring Щ„ЩҗЩ…ЩҸШұШ§ЩӮЩҺШЁШ© Ш§Щ„ЩҖ traffic ЩҲЩҺШ§ЩғШӘЩҗШҙШ§ЩҒ Ш§Щ„Ш§ЩҗЩҶШӘЩҗЩҮШ§ЩғШ§ШӘ.

6.5 NSX Security Groups ЩҲЩҺ Tags

рҹ§  Security Groups

  • Static Membership: ШЈЩҺШ¶ЩҗЩҒ VMs ЩҠЩҺШҜЩҺЩҲЩҠЩ‘Ш§ЩӢ Щ„ЩҗЩ„ЩҖ Group.
  • Dynamic Membership: VMs ШӘЩҺЩҶШ¶ЩҺЩ…Щ‘ ШӘЩҗЩ„ЩӮШ§ШҰЩҠЩ‘Ш§ЩӢ ШЁЩҗЩҶШ§ШЎЩӢ Ш№ЩҺЩ„Щү criteria (Name pattern, Tag, OS type).
  • NSX Tags: key-value pairs ШӘЩҸШұЩҒЩҺЩӮ ШЁЩҗШ§Щ„ЩҖ VM. Щ…ЩҺШ«ЩҺЩ„Ш§ЩӢ: env=prod, tier=web, app=crm.
  • Ш§Щ„ЩӮЩҸЩҲЩ‘Ш©: Tags + Dynamic Groups = firewall rules ШӘЩҺШӘЩҺЩғЩҺЩҠЩҺЩ‘ЩҒ ШӘЩҗЩ„ЩӮШ§ШҰЩҠЩ‘Ш§ЩӢ Щ…ЩҺШ№ ШӘЩҺШәЩҺЩҠЩҸЩ‘Шұ Ш§Щ„ШЁЩҠШҰШ©.

рҹ’Ў Щ…ЩҺШ«ЩҺЩ„ Ш№ЩҺЩ…ЩҺЩ„ЩҠ

6.6 vDefend вҖ” Ш§Щ„ШӯЩҗЩ…Ш§ЩҠШ© Ш§Щ„Щ…ЩҸШӘЩҺЩӮЩҺШҜЩҗЩ‘Щ…Ш©

6.8 Knowledge Check

Q1. Where does the Distributed Firewall (DFW) run?

Correct: B

DFW runs in the kernel of each ESX host, inspecting all traffic at the vNIC level.

Q2. What is the recommended default DFW rule action in a Zero-Trust model?

Correct: C

In Zero-Trust, the default rule should be Drop вҖ” only explicitly allowed traffic passes.

Q3. Which firewall type handles North-South traffic at the gateway level?

Correct: B

Gateway Firewall runs on Tier-0/Tier-1 Gateways and handles North-South traffic.

Q4. How do NSX Tags enable dynamic security groups?

Correct: B

NSX Tags are key-value pairs attached to VMs. Dynamic groups use tag-based criteria вҖ” VMs matching the criteria auto-join and firewall rules apply immediately.

Q5. Which DFW rule category has the highest priority?

Correct: C

Emergency rules have the highest priority, processed before all other categories.

Q6. Which component provides IDS/IPS and Malware Prevention in NSX?

Correct: C

vDefend (formerly NSX Advanced Threat Prevention) provides IDS/IPS, NTA, and Malware Prevention. It requires a separate license.

TK.2 Anti-Patterns вҖ” ШЈЩҺШ®Ш·Ш§ШЎ ЩӮШ§ШӘЩҗЩ„Ш©

1. The Flat Network Lover

В«ШЈЩҸШұЩҠШҜ ЩғЩҸЩ„Щ‘ VMs Ш№ЩҺЩ„Щү ЩҶЩҺЩҒШі VLAN вҖ” ШЈЩҺШЁШіЩҺШ·В» вҖ” ЩҮЩҺШ°Ш§ ЩҠЩҺЩҶЩҒЩҠ ЩғЩҸЩ„Щ‘ ЩҒЩҲШ§ШҰЩҗШҜ NSX. ШЁЩҗШҜЩҲЩҶ Overlay segments ЩҲЩҺ micro-segmentationШҢ ШЈЩҺЩҶШӘ ШӘЩҺШіШӘЩҺШ®ШҜЩҗЩ… NSX ЩғЩҺЩҖ expensive switch. ЩӮШ§Ш№ЩҗШҜШ©: ЩғЩҸЩ„Щ‘ Tier ШӘЩҺШ·ШЁЩҠЩӮЩҠ Ш№ЩҺЩ„Щү Segment Щ…ЩҸЩҶЩҒЩҺШөЩҗЩ„.

2. The MTU Ignorer

В«MTU 1500 ЩғШ§ЩҒЩҚ вҖ” Щ„ЩҗЩ…Ш§Ш°Ш§ ШЈЩҸШ№ЩҺЩӮЩҗЩ‘ШҜ Ш§Щ„ШЈЩҸЩ…ЩҲШұВ» вҖ” Geneve ЩҠЩҸШ¶ЩҠЩҒ ~50 bytes overhead. MTU 1500 вҶ’ fragmentation вҶ’ performance drops 30-50%. ЩӮШ§Ш№ЩҗШҜШ©: MTU 9000 end-to-end Щ„ЩҗЩғЩҸЩ„Щ‘ ШҙЩҺЩҠШЎ ЩҠЩҺШӯЩ…ЩҗЩ„ Geneve traffic.

3. The Active-Active Optimist

В«Active-Active ШЈЩҺЩҒШ¶ЩҺЩ„ ШЈЩҺШҜШ§ШЎ вҖ” ШіЩҺШЈЩҸШ¶ЩҠЩҒ NAT Щ„Ш§ШӯЩҗЩӮШ§ЩӢВ» вҖ” Щ„Ш§ ЩҠЩҸЩ…ЩғЩҗЩҶ ШҘЩҗШ¶Ш§ЩҒШ© NAT ШЈЩҺЩҲ VPN Щ„Щҗ Active-Active Tier-0. ЩҠЩҺШ¬ЩҗШЁ ШҘЩҗШ№Ш§ШҜШ© ЩҶЩҺШҙШұ Edge Cluster. ЩӮШ§Ш№ЩҗШҜШ©: ШҘЩҗШ°Ш§ ЩғЩҸЩҶШӘ ШәЩҺЩҠШұ Щ…ЩҸШӘЩҺШЈЩҺЩғЩҗЩ‘ШҜ вҶ’ Active-Standby.

4. The Single Manager

В«NSX Manager ЩҲШ§ШӯЩҗШҜ ЩғШ§ЩҒЩҚ Щ„ЩҗЩ„ШҘЩҗЩҶШӘШ§Ш¬ вҖ” ШіЩҺШЈЩҸШ¶ЩҠЩҒ nodes Щ„Ш§ШӯЩҗЩӮШ§ЩӢВ» вҖ” Manager ЩҲШ§ШӯЩҗШҜ = Щ„Ш§ HA. ЩҒЩҺШҙЩҺЩ„ЩҮ = ЩғЩҸЩ„Щ‘ NSX management ЩҠЩҺШӘЩҺЩҲЩҺЩӮЩҺЩ‘ЩҒ. ЩӮШ§Ш№ЩҗШҜШ©: 3-node cluster ШҜШ§ШҰЩҗЩ…Ш§ЩӢ ЩҒЩҠ Ш§Щ„ШҘЩҗЩҶШӘШ§Ш¬.

5. The Manual Router

В«Static Routing ШЈЩҺШЁШіЩҺШ· вҖ” Щ„Ш§ ШЈЩҺШӯШӘШ§Ш¬ BGPВ» вҖ” ЩҒЩҠ ШЁЩҠШҰШ© VCF Щ…ЩҺШ№ Ш№ЩҗШҜЩ‘Ш© Workload DomainsШҢ Static Routing Щ„Ш§ ЩҠЩҺШӘЩҺЩҲЩҺШіЩҺЩ‘Ш№. ЩғЩҸЩ„Щ‘ ШӘЩҺШәЩҠЩҠШұ ЩҠЩҺШӯШӘШ§Ш¬ ШӘЩҺШӯШҜЩҠШ« ЩҠЩҺШҜЩҺЩҲЩҠ. ЩӮШ§Ш№ЩҗШҜШ©: BGP Щ„ЩҗШЈЩҺЩҠЩ‘ ШЁЩҠШҰШ© ШЈЩҺЩғШЁЩҺШұ Щ…ЩҗЩҶ lab.

6. The "Allow All" Firewall

В«DFW Default Rule = Allow вҖ” ШЈЩҸШұЩҠШҜ ШЈЩҺЩҶ ШЈЩҺШӘЩҺШЈЩҺЩғЩҺЩ‘ШҜ ШЈЩҺЩҶЩҺЩ‘ ЩғЩҸЩ„Щ‘ ШҙЩҺЩҠШЎ ЩҠЩҺШ№Щ…ЩҺЩ„В» вҖ” ЩҮЩҺШ°Ш§ ЩҠЩҸЩ„ШәЩҠ ЩғЩҸЩ„Щ‘ ЩҒШ§ШҰЩҗШҜШ© Ш§Щ„ЩҖ DFW. ШЁЩҗШҜЩҲЩҶ Zero-Trust default вҶ’ lateral movement Щ…ЩҸЩ…ЩғЩҗЩҶ. ЩӮШ§Ш№ЩҗШҜШ©: Ш§ЩҗШЁШҜЩҺШЈ ШЁЩҗ Allow Щ„ЩҗЩ„ШӘЩҺЩ‘Ш№ЩҺЩ„ЩҸЩ‘Щ…ШҢ Ш«ЩҸЩ…ЩҺЩ‘ ШәЩҺЩҠЩҗЩ‘Шұ ШҘЩҗЩ„Щү Drop ЩҒЩҠ Ш§Щ„ШҘЩҗЩҶШӘШ§Ш¬.

7. The Backup Procrastinator

В«ШіЩҺШЈЩҸЩҮЩҺЩҠЩҗЩ‘ШҰ Ш§Щ„ЩҖ Backup ШәЩҺШҜШ§ЩӢВ» вҖ” ШәЩҺШҜШ§ЩӢ ЩӮЩҺШҜ ЩҠЩҺЩғЩҲЩҶ ШЁЩҺШ№ШҜ ЩҒЩҺЩҲШ§ШӘ Ш§Щ„ШЈЩҺЩҲШ§ЩҶ. ЩҒЩҺШҙЩҺЩ„ NSX Manager ШЁЩҗШҜЩҲЩҶ backup = ШҘЩҗШ№Ш§ШҜШ© ШЁЩҗЩҶШ§ШЎ ЩҠЩҺШҜЩҺЩҲЩҠЩ‘Ш© ЩғШ§Щ…ЩҗЩ„Ш©. ЩӮШ§Ш№ЩҗШҜШ©: ЩҮЩҺЩҠЩҗЩ‘ШҰ Scheduled Backup ЩҒЩҠ Ш§Щ„ЩҠЩҺЩҲЩ… Ш§Щ„ШЈЩҺЩҲЩҺЩ‘Щ„.

NSX Architecture

  • 3 Planes: Management, Control, Data
  • 3-node Manager Cluster (production)
  • Converged: Manager + Controller + Policy
  • VIP for API/UI access
  • Leader election + GARP on failure

Overlay Networking

  • Geneve encapsulation (UDP 6081)
  • TEP = Tunnel Endpoint (2 per node)
  • Overlay TZ + VLAN TZ
  • ESX вҶ’ VDS | Edge вҶ’ N-VDS
  • MTU minimum: 1700 (recommended 9000)

VPC Hierarchy

  • Default Space вҶ’ Projects вҶ’ VPCs вҶ’ Subnets
  • Transit GW per Project (auto-created)
  • VPC GW per VPC (auto-created)
  • Subnets: Public | Private-TGW | Private-VPC
  • Centralized vs Distributed connectivity

Tier-0 HA Modes

  • Active-Active: Routing + Reflexive NAT only
  • Active-Standby: NAT + VPN + Firewall + LB
  • Active-Standby required for: VCF Automation, Supervisor, IPSec/L2 VPN
  • Cannot change mode after deployment!

Edge Cluster

  • Max 10 nodes per cluster
  • 1 Tier-0 per Edge Node
  • 5 interfaces: eth0 + 4Г— fp-ethX
  • VCF auto-deploys Edge VMs
  • BGP default (OSPF supported)

Networking Services

  • NAT: SNAT + DNAT + Reflexive (Centralized only)
  • DHCP: Server | Relay (Public subnets only) | None
  • DNS Forwarder: on Tier-0, caches responses
  • LB: Tier-1 only, Active-Standby Edge
  • VPN: IPSec (L3) + L2 VPN вҖ” both Active-Standby

NSX Security

  • DFW: kernel-level on every ESX host (East-West)
  • GFW: on Tier-0/Tier-1 Gateways (North-South)
  • Rule order: Emergency вҶ’ Infra вҶ’ Env вҶ’ App вҶ’ Default
  • Tags + Dynamic Groups = auto-adapting policies
  • vDefend: IDS/IPS + NTA + Malware (separate license)

Segments & Profiles

  • Overlay Segment (Geneve) | VLAN Segment | VPC Subnet
  • QoS Profile: rate limiting + CoS/DSCP
  • IP Discovery: ARP/ND/DHCP Snooping
  • SpoofGuard: prevents IP/MAC spoofing
  • Segment Security: BPDU Filter, DHCP Block, RA Guard

Troubleshooting

  • get cluster status вҶ’ Manager health
  • vmkping ++netstack=vxlan -d -s 8972 вҶ’ TEP test
  • get bgp neighbor вҶ’ BGP status
  • API logs: /var/log/proton/nsxapi.log
  • 80% overlay issues = MTU mismatch!

Lifecycle Management

  • Backup вҶ’ SFTP server (scheduled + manual)
  • Restore: same version only, fresh cluster first
  • Upgrade order: Managers вҶ’ Edges вҶ’ Hosts
  • VCF upgrades via SDDC Manager only!
  • Certificates: plan renewal 30 days before expiry
